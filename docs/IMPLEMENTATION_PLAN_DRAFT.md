# АБ Афиша — план реализации (черновик)

> **Статус:** черновик. Порядок этапов может меняться.
> **Ключевое правило: код начинается только после Этапа 4.**

---

## Этап 1 — Финализировать дизайн

**Цель:** получить готовый к разработке дизайн.

- [ ] Завершить WIP-экран 1920 (`АБ Афиша main`).
- [ ] Утвердить визуальное направление (цвета, шрифты, иконки, стиль карточек).
- [ ] Проработать все основные экраны: главная, фильтры, карточка события, модальное окно.
- [ ] Утвердить ImportantEvents-секцию.
- [ ] Утвердить Header и Footer.

**Результат:** финальные экраны в Figma, согласованные со стейкхолдерами.

---

## Этап 2 — Утвердить компоненты и адаптивы

**Цель:** зафиксировать компонентный состав и поведение на всех устройствах.

- [ ] Описать компонентную декомпозицию (Header, FilterCard, EventCard, Modal и т.д.).
- [ ] Разработать дизайн для 1440, 1024, 390.
- [ ] Утвердить поведение фильтров на мобильном.
- [ ] Утвердить поведение модального окна на мобильном.
- [ ] Проработать все состояния: loading, empty, error, success.
- [ ] Обновить [DESIGN_HANDOFF.md](./DESIGN_HANDOFF.md) и [ADAPTIVE_RULES_DRAFT.md](./ADAPTIVE_RULES_DRAFT.md).

**Результат:** полный adaptive-дизайн, готовый к хендоффу.

---

## Этап 3 — Утвердить data model

**Цель:** зафиксировать структуру данных и ответить на открытые вопросы.

- [ ] Ответить на вопросы из [OPEN_QUESTIONS.md](./OPEN_QUESTIONS.md).
- [ ] Финализировать поля сущности Event.
- [ ] Определить связанные сущности: Category, City, Partner, Speaker.
- [ ] Решить вопрос регистрации (внешняя ссылка vs внутренняя форма).
- [ ] Определить интеграции: Telegram, экспорт в календарь.
- [ ] Обновить [DATA_MODEL_DRAFT.md](./DATA_MODEL_DRAFT.md).

**Результат:** утверждённая модель данных, готовая для написания Prisma schema.

---

## Этап 4 — Написать финальное ТЗ

**Цель:** зафиксировать полные требования перед стартом разработки.

- [ ] Описать все функциональные требования.
- [ ] Описать нефункциональные требования (производительность, SEO, безопасность).
- [ ] Зафиксировать API-контракты (если применимо).
- [ ] Зафиксировать роли и права доступа.
- [ ] Описать сценарии интеграций.
- [ ] Согласовать и подписать ТЗ со стейкхолдерами.

**Результат:** финальное ТЗ, утверждённое всеми сторонами.

---

## Этап 5 — Начать разработку

**Цель:** реализация продукта строго по финальному ТЗ и дизайну.

- [ ] Настроить проект и инфраструктуру.
- [ ] Реализовать Prisma schema и миграции.
- [ ] Реализовать бэкенд API.
- [ ] Реализовать фронтенд компоненты.
- [ ] Настроить интеграции.
- [ ] Провести QA.
- [ ] Запустить.

---

> До завершения Этапа 4 любой написанный код является экспериментальным и не может считаться частью продукта.
