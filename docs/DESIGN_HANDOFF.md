## Структура экрана

### Header

- Логотип / бренд АБ.
- Навигация (пункты TBD).
- Возможно: кнопка входа/профиля (зависит от решения по авторизации).

### Hero

- Заголовок афиши.
- Краткое описание или подзаголовок.
- Возможно: акцентный баннер или ключевое событие.

### FilterCard

- Фильтры по: дате, категории, формату (онлайн/офлайн/гибрид), городу, цене (бесплатно/платно).
- Поиск по названию (TBD).
- Сброс фильтров.
- На мобильном: фильтры скрыты под кнопкой или шторкой (см. ADAPTIVE_RULES_DRAFT.md).

### CalendarBlock

- Мини-календарь или хронологический список для навигации по датам.
- Выделение дней с событиями.
- Выбор даты — обновляет список карточек.

### EventCards

- Список или сетка карточек мероприятий.
- Карточка содержит: дата, название, формат, город, цена, превью-изображение, короткое описание.
- Клик по карточке — открывает EventDetailsModal.
- Пагинация или бесконечная прокрутка (TBD).

### ImportantEvents

- Отдельная секция или визуальный акцент для важных/ключевых событий.
- Выделяется из общего потока карточек.

### EventDetailsModal

- Полная информация о событии: название, дата/время, формат, город, место, цена, спикеры, описание, изображение.
- Кнопка регистрации / ссылка на регистрацию.
- Кнопки шаринга.
- Добавление в календарь (TBD).
- Кнопка Telegram-напоминания (TBD).
- Закрытие: крестик, клик по оверлею, Escape.

### States

Для каждого блока необходимо проработать состояния:

| Состояние | Описание |
|-----------|----------|
| Loading | Скелетон или спиннер во время загрузки данных |
| Empty | Нет событий по выбранным фильтрам — пустой стейт с подсказкой |
| Error | Ошибка загрузки — понятное сообщение, без mock-данных |
| Success | Нормальное отображение данных |
