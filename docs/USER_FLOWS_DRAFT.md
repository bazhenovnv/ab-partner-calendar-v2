# АБ Афиша — пользовательские сценарии (черновик)

> **Статус:** черновик. Сценарии не финализированы, некоторые шаги зависят от открытых вопросов (см. [OPEN_QUESTIONS.md](./OPEN_QUESTIONS.md)).

---

## Публичные сценарии (неавторизованный пользователь)

### 1. Пользователь открывает афишу

1. Пользователь переходит на страницу АБ Афиша.
2. Загружается список ближайших опубликованных событий.
3. Отображаются FilterCard, CalendarBlock и EventCards.
4. Если событий нет — показывается empty state с подсказкой.

### 2. Пользователь выбирает дату

1. Пользователь кликает на дату в CalendarBlock.
2. Список событий фильтруется по выбранной дате.
3. Если нет событий на эту дату — показывается empty state.
4. Пользователь может сбросить выбор даты.

### 3. Пользователь применяет фильтры

1. Пользователь выбирает один или несколько фильтров: категория, формат, город, цена.
2. Список событий обновляется по выбранным фильтрам.
3. Активные фильтры визуально выделены.
4. Кнопка «Сбросить» очищает все фильтры.

### 4. Пользователь открывает карточку события

1. Пользователь кликает на карточку события.
2. Открывается EventDetailsModal с полной информацией.
3. Пользователь видит: название, дату, время, формат, место, спикеров, описание, цену.

### 5. Пользователь переходит на регистрацию

1. В EventDetailsModal пользователь нажимает кнопку регистрации.
2. Если регистрация внешняя — открывается новая вкладка с сайтом организатора.
3. Если регистрация внутренняя (TBD) — открывается форма или модальное окно.

### 6. Пользователь смотрит все события

1. Пользователь сбрасывает все фильтры или нажимает «Все события».
2. Отображается полный список опубликованных событий.
3. Пагинация или бесконечная прокрутка (TBD).

### 7. Пользователь делится событием

1. Пользователь нажимает кнопку шаринга в EventDetailsModal или на карточке.
2. Возможные варианты: скопировать ссылку, поделиться в Telegram, другие каналы (TBD).
3. Ссылка на событие формируется по slug: `/events/{slug}`.

### 8. Пользователь переходит в Telegram / MAX

1. В EventDetailsModal пользователь нажимает кнопку «Напомнить в Telegram» или «Открыть в MAX».
2. Открывается Telegram deep link или MAX (TBD).
3. Пользователь подписывается на напоминание (TBD).

## Сценарии администратора (TBD)

> Наличие и форма админки пока не определены. См. [OPEN_QUESTIONS.md](./OPEN_QUESTIONS.md).

### 9. Админ создаёт событие

1. Админ открывает форму создания события.
2. Заполняет поля: название, дата, время, формат, город, место, категория, описание, изображение, ссылки.
3. Сохраняет как черновик (`status: draft`).
4. Проверяет предпросмотр.

### 10. Админ редактирует событие

1. Админ открывает список событий.
2. Выбирает событие и открывает форму редактирования.
3. Вносит изменения, сохраняет.
4. Если событие уже опубликовано — изменения отображаются сразу.

### 11. Админ публикует событие

1. Админ открывает черновик события.
2. Нажимает «Опубликовать».
3. Статус меняется на `published`, событие становится видимым публично.

### 12. Админ отменяет событие

1. Админ открывает опубликованное событие.
2. Нажимает «Отменить событие» и подтверждает.
3. Статус меняется на `cancelled`.
4. Событие остаётся видимым с пометкой «Отменено».
