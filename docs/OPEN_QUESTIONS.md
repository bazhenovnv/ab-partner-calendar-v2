# АБ Афиша — открытые вопросы

> Этот файл фиксирует вопросы, которые необходимо решить **до начала реализации**.
> Ответы на них влияют на дизайн, архитектуру, модель данных и бэкенд.

---

## Авторизация и пользователи

- [ ] Будет ли авторизация на публичной части афиши?
- [ ] Если да — через что: email, телефон, Telegram, OAuth (Google)?
- [ ] Нужны ли пользовательские профили или личный кабинет?
- [ ] Нужна ли история просмотренных / сохранённых событий?

## Создание и управление событиями

- [ ] Кто создаёт события: только внутренние редакторы или партнёры тоже?
- [ ] Если партнёры — нужна ли отдельная роль с ограниченным доступом?
- [ ] Нужна ли модерация перед публикацией событий от партнёров?
- [ ] Нужна ли возможность импорта событий из внешних источников (RSS, API, парсинг)?

## Регистрация на события

- [ ] Регистрация будет внутри сайта или только внешняя ссылка?
- [ ] Если внутренняя — нужна ли форма с полями (имя, email, телефон)?
- [ ] Нужно ли хранить список зарегистрировавшихся?
- [ ] Нужны ли email-подтверждения после регистрации?

## Напоминания и уведомления

- [ ] Нужны ли напоминания о событиях?
- [ ] Через какие каналы: Telegram, email, push?
- [ ] Напоминание за сколько времени до события?
- [ ] Пользователь сам подписывается или автоматически при регистрации?

## Telegram / MAX

- [ ] Нужна ли Telegram-интеграция (бот, канал, рассылка)?
- [ ] Нужна ли интеграция с MAX?
- [ ] Что именно в Telegram: напоминания, анонсы, кнопки регистрации?

## Экспорт в календарь

- [ ] Нужен ли экспорт события в Google Calendar / Apple Calendar / Outlook?
- [ ] Формат: .ics-файл или Add to Calendar-ссылка?

## Фильтры и справочники

- [ ] Нужна ли фильтрация по городам?
- [ ] Если да — будет ли справочник городов или свободный ввод?
- [ ] Нужна ли фильтрация по категориям? Кто управляет справочником категорий?
- [ ] Нужна ли фильтрация по спикерам или организаторам?
- [ ] Нужен ли поиск по тексту?

## Партнёры и спикеры

- [ ] Нужны ли отдельные страницы/профили партнёров?
- [ ] Нужны ли отдельные страницы/профили спикеров?
- [ ] Или партнёр/спикер — просто текстовое поле на событии?

## Административная панель

- [ ] Нужна ли отдельная административная панель?
- [ ] Или управление событиями будет через существующую систему (например, текущая AdminPanel)?
- [ ] Какие роли нужны в админке: суперадмин, редактор, партнёр?

## Аналитика и метрики

- [ ] Нужно ли отслеживать просмотры событий?
- [ ] Нужна ли аналитика переходов на регистрацию?
- [ ] Интеграция с Яндекс Метрикой / GA?

## Технические вопросы

- [ ] Какой стек будет использоваться (если отличается от существующего проекта)?
- [ ] Афиша — отдельное приложение или часть текущего монорепо?
- [ ] Нужна ли SEO-оптимизация (SSR/SSG для страниц событий)?
- [ ] Нужны ли OG-теги для шаринга в соцсетях?
