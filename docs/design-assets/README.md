# AB Афиша Бухгалтера — дизайн-материалы и эскизы

Эта папка фиксирует состав дизайн-материалов проекта для Claude/разработчиков.

## Главный эталон дизайна

**Основной утверждённый макет:**

```text
АБ Афиша main(4).pdf
```

Перед любыми изменениями публичной главной страницы Claude обязан сверяться с этим макетом и правилом **Design Freeze v1.0**.

## Что найдено в текущей сессии

В рабочем архиве собраны 167 дизайн-файлов:

- PDF-макеты: `АБ Афиша main(1).pdf`, `АБ Афиша main(2).pdf`, `АБ Афиша main(3).pdf`, `АБ Афиша main(4).pdf`;
- фреймы/логотипные материалы: `Frame 60.png`, `Frame 60(1).png`;
- рендер основного PDF-макета: `render_main4/page-1.png`;
- изображения maintenance/UI-эскизов: `a_*.png`;
- архивы креативов: `Креативы АБ (*.zip)`;
- загруженные изображения и скриншоты из обсуждений: `{GUID}.png`, `image(*).png`, `ChatGPT Image ...png`.

Полный перечень — в файле:

```text
docs/design-assets/DESIGN_ASSETS_MANIFEST.csv
```

## Где должны лежать бинарные файлы после загрузки в репозиторий

Рекомендуемая структура:

```text
project-assets/design/
├── 01_pdf_mockups/
├── 02_render_previews/
├── 03_logo_frames/
├── 04_creative_archives/
├── 05_generated_ui_mockups/

```

## Важно

В этой сессии через GitHub-коннектор можно безопасно записывать текстовые файлы, но не полноценные бинарные PDF/PNG/ZIP как рабочие файлы. Поэтому в GitHub добавлен manifest и README, а полный бинарный пакет подготовлен отдельным архивом:

```text
AB_Afisha_DESIGN_ASSETS_FOR_CLAUDE.zip
```

Его нужно загрузить в репозиторий через обычный git/браузер GitHub или передать Claude Code для распаковки в `project-assets/design/`.

## Правило для Claude

1. Не менять публичную главную страницу без сверки с `АБ Афиша main(4).pdf`.
2. Не добавлять новые маркетинговые блоки поверх утверждённого дизайна.
3. Не использовать термин «Важные события» — только «Главные события».
4. Не использовать «Проведено» — только «Завершено».
5. Все новые UI-правки встраивать в утверждённый Design Freeze v1.0.
