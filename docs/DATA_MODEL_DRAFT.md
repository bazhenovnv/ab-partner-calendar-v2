# АБ Афиша — черновая модель данных

> **Статус:** черновик. Это не Prisma schema и не миграция. Архитектура не закреплена.
> Поля, типы и связи могут измениться после финализации дизайна и ТЗ.

---

## Event (Событие)

| Поле | Тип (черновой) | Описание |
|------|----------------|----------|
| `id` | UUID / CUID | Уникальный идентификатор события |
| `title` | string | Название события |
| `slug` | string | URL-friendly идентификатор (уникальный) |
| `date` | date | Дата проведения |
| `startTime` | time / datetime | Время начала |
| `endTime` | time / datetime | Время окончания (опционально) |
| `city` | string / FK | Город (опционально для онлайн) |
| `location` | string | Место проведения или ссылка на онлайн-площадку |
| `format` | enum | `online` / `offline` / `hybrid` |
| `priceType` | enum | `free` / `paid` |
| `price` | number / null | Стоимость (null если бесплатно) |
| `category` | string / FK | Категория события |
| `image` | string / url | Обложка события |
| `shortDescription` | string | Краткое описание для карточки |
| `fullDescription` | text | Полное описание для модального окна |
| `registrationUrl` | url / null | Ссылка на регистрацию (внешняя или внутренняя — TBD) |
| `sourceUrl` | url / null | Ссылка на первоисточник |
| `partner` | string / FK / null | Организатор / партнёр |
| `speaker` | string / FK[] / null | Спикер(ы) (опционально) |
| `status` | enum | `draft` / `published` / `archived` / `cancelled` |
| `isImportant` | boolean | Помечено как важное/ключевое событие |
| `createdAt` | datetime | Дата создания записи |
| `updatedAt` | datetime | Дата последнего обновления |

---

## Примечания к полям

- **`slug`** — должен быть устойчивым; менять после публикации нежелательно (ломает ссылки).
- **`city`** — нужно ли вести справочник городов или хранить строкой, зависит от фильтров (TBD, см. [OPEN_QUESTIONS.md](./OPEN_QUESTIONS.md)).
- **`category`** — аналогично: справочник или enum (TBD).
- **`partner`** / **`speaker`** — отдельные сущности или встроенные строки (TBD).
- **`registrationUrl`** — если регистрация будет внутренней, потребуется отдельная модель Registration.
- **`price`** — валюта и формат отображения пока не определены.
- **`status: cancelled`** — отменённые события остаются в БД для корректного отображения ("отменено").
