# Frontend rules

## Общие правила

- Дизайн менять только по задаче.
- Если есть макет или скриншот — строго следовать ему.
- Не менять визуальную систему без согласования.
- Не менять пользовательские тексты без отдельной задачи.
- Не ломать адаптивность.
- Nullable-поля обрабатывать безопасно через optional chaining или явные проверки.

## Админка

- Не показывать mock-данные при ошибке API.
- При ошибке загрузки показывать понятное сообщение.
- Действия удаления, восстановления и публикации должны быть явно подтверждаемыми или визуально понятными.
- Не скрывать ошибки сохранения данных.

## Публичный календарь

- Сохранять текущий UX фильтров, карточек, модальных окон и важных событий.
- Не менять порядок и смысл блоков без отдельной задачи.
- Не ломать экспорт в календарь, Telegram-напоминания и ссылки на источники.
